## 修改需求

### 需求：答题状态管理

系统 MUST 通过 QuestionProvider 管理答题流程状态，**并在答错时自动记录到错题本**。

#### 场景：答错自动记录

- **给定** 用户正在答题
- **当** 用户选择错误选项并提交
- **那么** 判断答案错误
- **并且** 自动调用 `wrongBookRepository.addWrongBook(questionId)`
- **并且** 显示反馈面板提示已加入错题本

#### 场景：错题复习模式

- **给定** 用户从错题本进入复习
- **当** 调用 `startWrongBookReview()`
- **那么** 加载错题本中的题目
- **并且** 答题流程与正常模式一致
- **并且** 答对后提示是否标记为已掌握

#### 场景：记录答题历史

- **当** 每次提交答案后
- **那么** 调用 `answerRepository.recordAnswer()` 保存答题记录
- **并且** 更新本地学习进度统计

## 新增需求

### 需求：系统 MUST 支持错题本复习流程

系统 MUST 提供专门的错题复习流程，帮助用户针对性强化薄弱环节。

#### 场景：启动错题复习

- **给定** 用户在错题本页面
- **当** 点击"开始复习"按钮
- **那么** 进入错题复习模式
- **并且** 只显示未掌握的错题

#### 场景：错题复习完成

- **给定** 用户完成所有错题复习
- **当** 最后一题答对
- **那么** 显示复习完成统计
- **并且** 提供"标记全部掌握"选项
