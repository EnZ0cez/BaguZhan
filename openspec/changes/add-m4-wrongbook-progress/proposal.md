# 变更：添加错题本与学习进度持久化 (M4)

## 为什么
M1-M3 已完成基础答题循环和 UI 优化，但用户的学习数据（错题、答题历史、进度）仅在内存中，应用重启后丢失。M4 需要实现数据持久化，让用户能够：
- 回顾和复习做错的题目
- 追踪学习进度和统计数据
- 跨会话保持学习连续性

## 变更内容
- **新增** 错题本数据模型和存储（PostgreSQL + 本地缓存）
- **新增** 用户答题历史记录
- **新增** 学习进度统计（每日/每周/总体）
- **新增** 错题本 UI 页面（列表、详情、复习模式）
- **新增** 学习报告页面
- **新增** BFF API 端点（错题 CRUD、进度查询）
- **修改** 答题流程，答错时自动加入错题本

## 影响
- 受影响规范：database, bff, flutter-app, question-flow
- 受影响代码：
  - `server/src/` - 新增错题服务和 API
  - `baguzhan/lib/data/` - 新增错题仓库
  - `baguzhan/lib/presentation/` - 新增错题本页面
- **重大变更**：数据库 Schema 新增表（user_answers, wrong_book, learning_progress）
